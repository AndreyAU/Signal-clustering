# Signal Clustering: Pulse Shape Discrimination

# Классификация сигналов ядерных излучений методом машинного обучения 

---

## Описание проекта

Проект посвящён кластеризации сигналов сцинтилляционного детектора на три группы:

- `cluster 0` — первый тип частиц;
- `cluster 1` — второй тип частиц;
- `cluster 2` — неоднозначные, смешанные или аномальные сигналы.

Задача решалась как задача обучения без учителя: в исходных данных нет известных меток классов, поэтому на этапе исследования сравнивались разные способы описания сигналов и разные алгоритмы кластеризации.

Основная идея финального решения — использовать не только амплитуду и площадь импульса, а форму раннего хвоста после максимума. Такой подход соответствует логике Pulse Shape Discrimination: сигналы разных типов частиц могут отличаться временем и формой спада.

## Ход исследования

Исследование проводилось в несколько этапов.

### 1. Сначала была выполнена базовая предобработка сигналов:
- отделение первых 4 metadata-колонок;
- расчёт индивидуальной базовой линии по первым 50 отсчётам;
- инверсия сигнала относительно baseline;
- формирование двух представлений:
- signals_positive — для расчёта площади и энергетических признаков;
- signals_inverted — для анализа формы сигнала без обрезки значений около нуля.

### 2. Далее были проверены разные группы признаков:
- энергетические признаки: амплитуда, площадь, sqrt(volume), log(volume);
- геометрические признаки: отношение амплитуды к площади, угол, эксцесс;
- PSD-признаки;
- PCA-признаки хвоста после максимума.

### 3. На физико-интерпретируемых признаках были протестированы:
- KMeans;
- GaussianMixture;
- AgglomerativeClustering.

По внутренним метрикам хорошо выглядели модели на энергетических признаках. Однако анализ профилей кластеров показал, что они в основном разделяют сигналы по мощности импульса: слабые, сильные и очень сильные. Такое разделение не полностью соответствует физической постановке задачи.

### 4. После этого был отдельно исследован tail-подход: 
из каждого сигнала извлекался короткий хвост после максимума, затем хвост нормализовался, масштабировался, понижался через PCA и кластеризовался через KMeans.

Важное улучшение: хвост для финальной модели извлекался из signals_inverted, а не из signals_positive. Это позволило не обрезать небольшие отрицательные колебания около baseline и сохранить реальную форму раннего спада.

## Финальная модель

Финальная модель использует форму раннего хвоста сигнала после максимума.

Параметры финального пайплайна:

| Этап                  | Значение                                            |
| --------------------- | --------------------------------------------------- |
| Baseline window       | 50 первых отсчётов                                  |
| Представление сигнала | `signals_inverted`                                  |
| Длина хвоста          | `tail_len = 19`                                     |
| Нормализация хвоста   | деление на собственный максимум                     |
| Масштабирование       | `StandardScaler`                                    |
| Понижение размерности | `PCA(n_components=0.90)`                            |
| Кластеризация         | `KMeans(n_clusters=3, n_init=500, random_state=42)` |

Финальный подход оказался лучше моделей, основанных только на амплитуде и площади, потому что он учитывает форму раннего спада импульса. Это важнее для разделения сигналов разных типов, чем простой энергетический масштаб.

На локальной настройке были проверены близкие варианты tail_len и доли объяснённой дисперсии PCA. Два варианта дали одинаковый public score:
| Вариант                       | Public score |
| ----------------------------- | -----------: |
| `tail_len = 19`, `PCA = 0.90` |    `0.84807` |
| `tail_len = 19`, `PCA = 0.91` |    `0.84807` |

Финальным выбран вариант PCA = 0.90, так как он использует более компактное представление хвоста и не ухудшает качество.

Финальный submission:
`submission_tail_pca_19_inverted_signal_pca_09_kmeans_n500.csv`

Лучший public score на Kaggle:
`0.84807`
